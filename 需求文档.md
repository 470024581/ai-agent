# Smart AI Assistant 需求文档

## 1. 项目概述

### 1.1 项目背景
Smart AI Assistant 是一个企业级智能数据分析平台，结合了自然语言处理与先进的工作流编排技术。该系统通过 React 前端界面与强大的 FastAPI 后端集成，具备 LangGraph 工作流编排、多 LLM 支持和实时数据处理能力。

**项目定位**: 面向企业用户的智能数据分析助手，降低数据分析门槛，提升业务决策效率。

**核心价值**: 
- 将复杂的数据库查询转换为自然语言交互
- 提供多模态数据源统一分析能力
- 实现 AI 驱动的智能洞察和建议
- 提供详细的执行过程追踪和调试能力

### 1.2 项目目标

#### 1.2.1 主要目标
- **智能化**: 提供基于自然语言的智能数据查询与分析
- **统一化**: 支持多种数据源类型的统一管理和分析
- **实时化**: 实现实时工作流执行监控和状态反馈
- **可视化**: 提供详细的工作流执行日志和节点推理过程展示
- **可扩展**: 构建可扩展的企业级 AI 应用架构

#### 1.2.2 业务目标
- 提升数据分析效率 80% 以上
- 降低非技术用户的数据分析门槛
- 支持快速业务决策和洞察发现
- 建立标准化的数据分析工作流
- 提供透明的AI决策过程，增强用户信任

#### 1.2.3 技术目标
- 构建高性能、高可用的系统架构
- 实现模块化、可插拔的功能设计
- 支持多环境部署和水平扩展
- 建立完善的监控和运维体系
- 提供详细的执行日志和调试信息

#### 1.3.2 技术栈详情
- **前端技术栈**:
  - React 18: 现代化 UI 框架，支持并发特性
  - Redux Toolkit: 状态管理，支持 RTK Query
  - Tailwind CSS: 原子化 CSS 框架

  - Pydantic: 数据验证和序列化
  - WebSocket: 实时通信
  - FAISS: 向量相似性搜索
  - Cross-Encoder: 文档重排序（ms-marco-MiniLM-L-6-v2）

- **AI 服务集成**:
  - OpenAI GPT-4/3.5: 主要语言模型
  - OpenRouter: 多模型 API 网关
  - Ollama: 本地模型部署
  - HuggingFace: 嵌入模型和开源模型
  - Cross-Encoder: 文档重排序模型

- **数据存储**:
  - SQLite: 开发环境数据库
  - PostgreSQL: 生产环境数据库
  - FAISS: 向量相似性搜索
  - 文件系统: 文档和媒体文件存储

## 2. 核心功能需求

### 2.1 智能问答系统

#### 2.1.1 自然语言查询
**功能描述**: 用户可以使用自然语言进行数据查询，系统自动理解用户意图并生成相应的分析结果。

**核心特性**:
- **多语言支持**: 支持中文、英文查询，自动识别语言类型
- **智能理解**: 基于 LLM 的意图识别和实体提取
- **上下文记忆**: 支持多轮对话，保持上下文连贯性
- **查询优化**: 自动优化查询语句，提高查询效率

**示例查询场景**:
- **销售分析**: "本月销售额多少？"、"哪个产品销量最好？"
- **库存管理**: "库存低于50的产品有哪些？"、"哪些产品需要补货？"
- **趋势分析**: "过去7天的销售趋势如何？"、"环比增长情况？"
- **客户分析**: "VIP客户有哪些？"、"客户购买行为分析？"
- **财务分析**: "利润率最高的产品类别？"、"成本分析报告？"

**技术实现**:
- 使用 LangGraph 工作流引擎处理复杂查询
- 集成多个 LLM 提供商确保服务稳定性
- 实现查询缓存机制提升响应速度
- 支持查询结果的质量评估和自动重试

#### 2.1.2 多数据源支持
**功能描述**: 系统支持多种类型的数据源，能够统一管理和分析不同类型的数据。

**数据源类型**:
- **知识库 (Knowledge Base)**:
  - 支持 PDF、Word、TXT 等文档格式
  - 基于 FAISS 的向量相似性搜索
  - 支持文档内容的语义理解和问答
  - 自动文档分块和索引建立
  - **检索策略**: 检索 Top 10 文档，使用 Cross-Encoder 重排序到 Top 3

- **SQL 表 (Structured Data)**:
  - 支持关系型数据库查询
  - 自动 SQL 生成和优化
  - 支持复杂的数据关联查询
  - 提供数据表结构自动发现

- **混合模式 (Hybrid)**:
  - 结合结构化数据和非结构化文档
  - 支持跨数据源的关联分析
  - 提供综合性的分析结果
  - 智能选择最适合的数据源

**数据源管理**:
- 动态数据源配置和切换
- 数据源健康状态监控
- 数据源权限和安全控制
- 数据源性能优化和缓存

#### 2.1.3 LangGraph 工作流
**功能描述**: 基于 LangGraph 的智能工作流编排，实现多步骤的AI处理流程。

**工作流节点**:
- **RAG Query Node (RAG查询节点)**:
  - 从知识库检索 Top 10 相关文档
  - 使用 Cross-Encoder 重排序到 Top 3
  - 基于重排序后的文档生成答案
  - 记录文档来源、相似度分数和内容预览
  - 显示检索、重排序、答案生成的详细过程

- **Router Node (路由节点)**:
  - 分析用户查询和RAG答案
  - 判断是否需要SQL-Agent进行数据库查询
  - 提供路由决策的推理过程
  - 支持SQL-Agent路径和纯RAG路径

- **SQL Agent Node (SQL代理节点)**:
  - 基于ReAct模式的SQL探索
  - 自动生成和执行SQL查询
  - 跟踪中间步骤（工具调用、观察结果）
  - 显示执行的SQL语句和使用的表名
  - 展示查询结果的行数、列名和样例数据

- **Chart Process Node (图表处理节点)**:
  - 分析结构化数据是否适合图表生成
  - 自动选择合适的图表类型（柱状图、折线图、饼图等）
  - 生成图表配置和数据点
  - 显示图表生成决策和数据样例

- **LLM Processing Node (LLM处理节点)**:
  - 整合RAG答案、SQL结果、图表信息
  - 生成最终的自然语言答案
  - 支持流式输出和实时更新
  - 显示整合过程和生成内容

**工作流特性**:
- 实时执行状态追踪
- 节点级别的错误处理和重试
- 详细的执行日志记录
- 支持工作流中断和恢复

### 2.2 实时执行日志

#### 2.2.1 执行日志显示
**功能描述**: 提供详细的工作流执行日志，实时显示每个节点的处理过程和推理细节。

**日志内容**:
- **节点执行状态**:
  - 节点开始、进行中、完成、失败状态
  - 节点执行时间和性能指标
  - 节点输入输出数据预览

- **RAG查询节点日志**:
  - 检索到的文档数量（Top 10）和来源文件
  - 文档相似度分数和内容预览
  - 重排序后的Top 3文档及其Cross-Encoder分数
  - 答案生成过程和生成内容
  - 使用的源文档列表及完整路径

- **路由节点日志**:
  - RAG答案预览
  - 路由决策的推理过程
  - 最终决策（需要SQL-Agent/纯RAG路径）

- **SQL代理节点日志**:
  - ReAct循环步骤数量及工具名称
  - 每个中间步骤的工具输入和观察结果
  - 执行的SQL查询及使用的表名
  - SQL执行结果的行数、列名和样例数据
  - SQL Agent生成的自然语言答案

- **图表处理节点日志**:
  - 数据分析结果（数据点数量、字段信息）
  - 图表生成决策（是否适合、图表类型）
  - 图表配置（X/Y轴、数据点样例）

- **LLM处理节点日志**:
  - 输入源汇总（RAG、SQL、图表、结构化数据）
  - 整合过程说明
  - 流式答案生成过程

**日志特性**:
- **实时滚动**: 自动滚动到最新日志条目
- **颜色编码**: 不同类型日志使用不同颜色（成功/错误/进行中/推理/信息）
- **详细展示**: 显示文档来源、数据样例、SQL查询等详细信息
- **可读性**: 格式化显示，支持长文本分块展示

- **TypeScript**: 提供类型安全和更好的开发体验
- **Vite**: 快速构建工具，支持热重载和模块联邦
